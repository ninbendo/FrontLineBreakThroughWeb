# Day開始ルーチン

引数 `$ARGUMENTS` にDay番号（例: `1`, `3`）またはブランチ名が渡される。

## 手順

### 1. タスク計画の読み込み

`docs/mvp-task-plan-2026-03-16.md` を読み、該当Dayの情報を取得する:
- ブランチ名
- タスク一覧（タスクID・内容・区分）
- 見積もり工数
- 完了基準
- 動作確認テストID

### 2. ブランチ作成

- mainブランチから新しいブランチを作成して切り替える
- ブランチ名はタスク計画に記載されたものを使用する
- 例: `feature_task-4-1-soldier-system`

### 3. タスク概要の提示

以下をまとめて報告する:
- 今日のタスク一覧と完了基準
- 動作確認で使うテストID
- 関連する仕様書ファイル（`docs/specs/` 内）
- 見積もり工数

### 4. 実績トラッキング更新

`docs/mvp-task-plan-2026-03-16.md` の実績トラッキング表で、該当Dayの「予定日」が空欄の場合、今日の日付を記入する。
