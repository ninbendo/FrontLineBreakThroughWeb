## 根本的な問題（3件）

### 問題1: 兵隊システムが存在しない（最大の乖離）

**仕様の設計**:
```
PlayerGroupController（集団管理）
  ├─ currentSoldierCount（人数）
  ├─ internalCount（内部カウント、上限なし、スコア計算用）
  ├─ renderCap = 89（描画上限）
  └─ SoldierUnit[]（個体、各HP=2）
       └─ SoldierFormationController（中心+4リング配置: 6/12/20/50）
```

**現実装**:
```
PlayerGroupController
  ├─ currentHp = 2（グループ全体をHP=2の1体として扱う）
  └─ （兵隊人数・個体・配置の概念なし）
```

**影響範囲**: ゲート・タル・トゲ・アイテム・HUD全てがこの問題に依存

### 問題2: ゲートの仕組みが根本的に違う

**仕様**: 弾1発ごとにint値が±1変化 → 接触時にその時点の値で人数反映 → 色が動的変化（負数=赤、0以上=青）

**現実装**: 固定のeffectValueを持ち、接触でHeal/TakeDamageするだけ。弾との相互作用なし。

### ~~問題3: 座標系の不整合の可能性~~ → **解消済み（2026-03-17 PR#24）**

全オブジェクトをZ軸移動に統一。カメラを3D見下ろし斜め視点に配置。BarrelController/GateControllerにもZ軸移動を追加。
